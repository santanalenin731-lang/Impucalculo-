# Efficiency & Design Protocols

## 1. Communication Protocol (Save Tokens)
*   **Minimalist Responses:** No fluff, no greetings, no "I will do this". Just execution summaries.
*   **Direct Action:** State the file modified and the specific change.
*   **No Repetition:** Do not repeat the user's prompt.

## 2. Design Precision (Impucálculo Defaults)
*   **Colors:**
    *   Primary: `#0038A8` (Dr. Blue)
    *   Secondary: `#CE1126` (Dr. Red)
    *   Text: `#1E293B` (Slate 900)
    *   Bg: `#F8FAFC` (Slate 50)
*   **Typography:** 'Outfit', sans-serif.
*   **Components:**
    *   **Cards:** `glass rounded-[2rem] neo-shadow border-white`.
    *   **Buttons:** `rounded-xl btn-gradient-blue` or `bg-dr-blue`.
    *   **Inputs:** `rounded-2xl border-2 border-dr-slate-100`.
*   **Behavior:**
    *   Always assume "Modern/Premium" aesthetic.
    *   Use `animate-slide-up` and `fadeIn` for new elements.
    *   Icons always solid background, no gradients.

## 3. Workflow
*   **Precision:** Targeted code edits (replace only what's needed).
*   **Speed:** Anticipate next steps (e.g., if adding a button, add its CSS too).
